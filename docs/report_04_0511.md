# Siren System Status Report - 04/05/11

## Status Atual
O sistema está operacional, mas enfrentou problemas de persistência de dados que afetaram a busca de configurações de túnel.

## O Que Está Acontecendo?

### 1. Incompatibilidade de Unmarshaling (Resolvido)
O principal motivo do erro "configuração de túnel não encontrada" era uma falha do Viper ao carregar os campos do JSON para as structs Go. 
- O Viper utiliza a biblioteca `mapstructure`, que por padrão tenta casar nomes de campos de forma case-insensitive.
- Campos como `remote_device_id` (JSON) não eram mapeados automaticamente para `RemoteDeviceID` (Go) sem tags explícitas.
- **Ação**: Adicionamos tags `mapstructure` em todas as structs no arquivo `models.go`.

### 2. Bloqueio por Tipo de Dados (Resolvido)
Havia um erro crítico no carregamento da lista de dispositivos.
- O campo `LastSeen` estava definido como `time.Time`. O mapeador do Viper não consegue converter automaticamente strings de data (ex: "2026-05-10...") para `time.Time` sem um hook customizado.
- Isso fazia com que a função `GetDevices()` retornasse uma lista vazia, bloqueando qualquer operação subsequente.
- **Ação**: Alteramos o tipo de `LastSeen` para `string` e ajustamos o `Manager` para formatar a data corretamente.

### 3. Inconsistência em Dados Antigos
Dispositivos adicionados ANTES das correções acima podem ter sido salvos com inconsistências no arquivo `config.json`.
- Observamos entradas na lista de túneis com `remote_device_id` vazio (`""`).
- Isso acontece quando o salvamento falha em mapear o ID corretamente ou quando a leitura falha e o sistema tenta atualizar uma entrada que não carregou direito.

## Diagnóstico da Falha Reportada
Se o erro ainda persiste para um dispositivo específico:
1. Verifique se o ID passado no terminal existe no arquivo `~/.config/siren/config.json` tanto na lista de `devices` quanto na lista de `tunnels`.
2. Se o dispositivo existir em `devices` mas não em `tunnels`, o erro "configuração de túnel não encontrada" será exibido.
3. Para novos dispositivos adicionados agora, o sistema deve funcionar perfeitamente.

## Próximos Passos Recomendados
- **Limpeza**: Recomenda-se remover o arquivo de configuração antigo (`rm ~/.config/siren/config.json`) e cadastrar os dispositivos novamente para garantir a integridade total dos dados.
- **Robustez**: Implementar uma criação automática de configuração de túnel padrão caso ela não seja encontrada ao iniciar um túnel para um dispositivo válido.
