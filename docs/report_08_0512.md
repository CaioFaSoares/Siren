# Relatório de Requisitos - Interface Gráfica Siren (08/05/12)

Este relatório define os requisitos funcionais e estéticos para a futura interface gráfica (GUI) do **Siren**, utilizando o framework **Wails** com **Nuxt 3**.

## 1. Visão Geral e Estética
A interface deve transmitir uma sensação **premium**, profissional e moderna, focada em power users.
- **Tema**: Dark Mode nativo (estilo Tokyo Night ou similar).
- **Efeitos**: Uso de *Glassmorphism* (blur de fundo) e micro-animações suaves para transições.
- **Tipografia**: Fontes sem-serifa modernas (ex: Inter, Roboto ou Outfit).
- **Layout**: Sidebar para navegação principal e área central dinâmica.

## 2. Requisitos Funcionais

### A. Dashboard (Visão Geral)
- Exibição de túneis ativos no momento.
- Status da conexão (IP, latência estimada, portas em uso).
- Botão de parada rápida (Emergency Stop) para encerrar todos os processos de áudio.

### B. Gerenciamento de Dispositivos (Inventário)
- Lista de dispositivos cadastrados com ícones representando a plataforma (macOS/Linux).
- Indicador de status (Online/Offline) via ping simples ou heartbeat.
- Formulário para adicionar/editar dispositivos (Nome, IP, Plataforma).

### C. Configuração de Túnel (O "Wizards")
- Fluxo guiado para iniciar um túnel:
    1. **Selecionar Destino**: Escolher um dispositivo do inventário.
    2. **Selecionar Modo**: Botões visuais para "Enviar Áudio", "Receber Áudio" ou "Duplex".
    3. **Roteamento de Nodes**: 
       - Dropdown para escolher o Microfone/Source local.
       - Dropdown para escolher o Alto-falante/Sink remoto (buscado via API do Core).
    4. **Configurações Avançadas**: Ajuste de portas (opcional, com defaults preenchidos).
    5. **Botão "Lançar"**: Feedback visual de progresso enquanto o motor de áudio inicia.

### D. Monitoramento em Tempo Real
- Visualização de níveis de áudio (VU Meter) para entrada e saída.
- Log de eventos simplificado para depuração (ex: "Conexão estabelecida", "Erro no PipeWire").

## 3. Integração Técnica (Wails Bind)
A UI deve se comunicar com o backend através dos métodos expostos na struct `App` em `app.go`:
- `GetDevices()`: Retorna a lista do store.
- `GetNodes(type)`: Retorna os dispositivos de áudio locais.
- `StartTunnel(config)`: Inicia o motor de áudio.
- `StopTunnel()`: Encerra o motor.

## 4. Experiência do Usuário (UX)
- **Feedback de Erro**: Notificações *Toast* claras quando um comando falha (ex: "Erro: PipeWire não encontrado").
- **State Management**: A UI deve refletir o estado real do backend. Se o túnel cair no backend, a UI deve mudar para o estado "Desconectado" automaticamente.
- **Responsividade**: Janela redimensionável com layout adaptativo.
