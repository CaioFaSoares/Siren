# Relatório de Estado do Projeto: Siren
**Data:** 10 de Maio de 2026
**Versão:** 0.1.0 (Initial Setup)

## 1. Visão Geral
O **Siren** é uma ferramenta de tunneling de áudio projetada para operar de forma híbrida, oferecendo tanto uma interface gráfica (GUI) quanto uma interface de linha de comando (CLI). O objetivo principal é permitir o roteamento de áudio entre dispositivos (ex: MacBook Pro e Yamori Linux).

## 2. Pilha Tecnológica
- **Backend:** Go (Golang)
- **Framework GUI:** Wails v2
- **Interface CLI:** Cobra
- **Frontend:** Nuxt.js (Vue 3 + TypeScript)
- **Ambiente de Desenvolvimento:** Nix (Flake)

## 3. Arquitetura e Estrutura de Arquivos

### Raiz do Projeto
- `main.go`: Ponto de entrada unificado. Gerencia a lógica CLI via Cobra e inicia a GUI via Wails.
- `app.go`: Lógica de integração do Wails (bindings entre Go e JS).
- `wails.json`: Configuração do framework Wails.
- `flake.nix`: Define as dependências e o ambiente de desenvolvimento reproduzível.

### Diretórios Principais
- `core/`: Contém a lógica de domínio e definições fundamentais.
  - `models.go`: Estruturas de dados para `Device` e `TunnelConfig`.
  - `engine.go`: Interface `Engine` para controle do túnel (Start/Stop/Status).
- `frontend/`: Aplicação Nuxt.js.
  - Utiliza o diretório `app/` para componentes e páginas.
  - Integrado ao Wails para comunicação bidirecional com o backend.
- `docs/`: Documentação do projeto (incluindo este relatório).
