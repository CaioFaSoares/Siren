# Siren - Status Report (05-12)

## Resumo das Atualizações
Este relatório detalha as correções críticas realizadas nos motores de áudio para macOS e Linux, além da implementação da passagem dinâmica de IP para os túneis.

## 1. Ajuste da Passagem de IP (Contexto Global)
Anteriormente, os motores de áudio utilizavam o IP `127.0.0.1` de forma hardcoded, o que impedia a comunicação real entre dispositivos na rede.

- **Interface `AudioEngine`**: O método `Start` agora recebe obrigatoriamente o `targetIP` como argumento.
- **Orquestrador (`Manager`)**: O método `StartTunnelToDevice` agora busca o IP real do dispositivo cadastrado no Store e o repassa para o motor de áudio.
- **Arquivos modificados**: `core/engine.go`, `core/manager.go`.

## 2. Correções no Engine macOS (Darwin)
Identificamos que o comando `roc-send` estava falhando devido ao uso de flags inexistentes ou sintaxe incorreta para a versão atual do ROC Toolkit.

- **Remoção da flag `-p`**: A flag `-p rs8m` causava o erro `invalid option -- p`.
- **Nova Sintaxe de URI**: O comando agora utiliza o formato URI correto para os protocolos de áudio e reparo:
  - `-s rtp+rs8m://<targetIP>:<port>`
  - `-r rs8m://<targetIP>:<port>`
- **Preservação de Input**: Mantivemos a captura via `coreaudio` para garantir que o microfone seja enviado.
- **Arquivo modificado**: `core/engine_darwin.go`.

## 3. Correções no Engine Linux (PipeWire)
O motor de áudio Linux apresentava falhas na captura do ID do módulo e na limpeza de recursos.

- **Sucesso Silencioso**: Ajustamos a lógica para que o Siren não retorne erro fatal caso o comando `pw-cli load-module` retorne uma saída vazia (comum em algumas versões do PipeWire/WirePlumber), desde que o processo retorne sucesso (`err == nil`).
- **Limpeza Robusta (`Stop`)**: Implementamos um fallback no método `Stop`. Caso o ID do módulo não tenha sido capturado no `Start`, o Siren agora executa um `pkill -f libpipewire-module-roc-sink` para garantir que o túnel seja encerrado de forma limpa.
- **IP Dinâmico**: O parâmetro `remote.ip` agora reflete o IP real do destino.
- **Arquivo modificado**: `core/engine_linux.go`.

## Status Atual
O sistema Siren agora é capaz de:
1. Cadastrar dispositivos com IPs reais (ZeroTier/Local).
2. Iniciar túneis de áudio funcionalmente em ambas as plataformas (macOS/Linux) apontando para o IP correto.
3. Encerrar túneis de forma segura, evitando processos órfãos ou módulos presos no PipeWire.
