    - Sistema de processo persistente (`pw-cli`) que mantém os módulos carregados no PipeWire enquanto o túnel estiver ativo, evitando que sumam do sistema (ex: Pavucontrol).
- **Integração macOS (ROC CLI)**:

## 3. Como o Sistema Funciona
O Siren utiliza o protocolo **ROC** (Real-time Over-the-network Communications), focado em áudio de alta qualidade e baixa latência com correção de erros (FEC).

### Fluxo no Linux
No Linux, o Siren interage diretamente com o **PipeWire**. Quando um túnel é iniciado, ele abre um pipe para o comando `pw-cli` e injeta comandos `load-module`. Enquanto esse pipe estiver aberto, o PipeWire mantém os módulos ativos. Isso cria interfaces virtuais de áudio no sistema que podem ser roteadas para qualquer app.

### Fluxo no macOS
No macOS, como o suporte nativo do ROC via CoreAudio é complexo, o Siren gerencia instâncias dos binários `roc-send` (para transmitir áudio do microfone) e `roc-recv` (para tocar o áudio recebido nos alto-falantes).

## 4. O "Ir e Vir" do Áudio (Duplex)
A questão de "ir e vir" do áudio foi resolvida com a implementação do **Modo Duplex**. 

### Como funciona a Bidirecionalidade:
Para evitar conflitos de rede e permitir o tráfego simultâneo, o Siren utiliza pares de portas distintos para cada direção:

1. **Direção A (Transmissão Local -> Remoto)**:
   - **Source Port**: 10001 (Padrão)
   - **Repair Port**: 10002 (Padrão)
   - *No Linux*: Carrega o `roc-sink`.
   - *No Mac*: Roda o `roc-send`.

2. **Direção B (Recepção Remota -> Local)**:
   - **RX Source Port**: 10003 (Padrão)
   - **RX Repair Port**: 10004 (Padrão)
   - *No Linux*: Carrega o `roc-source`.
   - *No Mac*: Roda o `roc-recv`.

Quando o modo **Duplex** é ativado, o Siren inicia ambos os processos/módulos simultaneamente. Isso permite, por exemplo, que você use o microfone do seu Mac no Linux e, ao mesmo tempo, ouça o som do Linux nos alto-falantes do Mac.

## 5. Conclusão
O sistema está "redondinho" no que tange à estabilidade do transporte de áudio. O uso do `pw-cli` persistente no Linux resolveu o problema de módulos "fantasmagóricos" que sumiam, e a lógica de portas separadas para Duplex garante que não haja colisão de pacotes RTP na rede.
