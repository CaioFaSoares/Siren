# Siren - Status Report (06-12) - Plano para Modo Receiver

## Objetivo
Transformar o Siren em uma ferramenta de áudio bidirecional, permitindo que cada dispositivo escolha se deseja **Enviar** (Sender) ou **Receber** (Receiver) fluxos de áudio ROC.

## 1. Mudanças nos Modelos (`core/models.go`)
Precisamos tipar os modos de operação para que o orquestrador e os motores saibam qual comando executar.

```go
type TunnelMode string

const (
	ModeSender   TunnelMode = "sender"   // Envia áudio local para remoto
	ModeReceiver TunnelMode = "receiver" // Recebe áudio remoto para local
)

type TunnelConfig struct {
	// ... campos existentes ...
	Mode TunnelMode `json:"mode" mapstructure:"mode"`
}
```

## 2. Implementação no macOS (`core/engine_darwin.go`)
Atualmente apenas o `roc-send` está implementado.

- **Novo Comando**: `roc-recv`.
- **Sintaxe**: 
  - `roc-recv -s rtp+rs8m://0.0.0.0:<port> -r rs8m://0.0.0.0:<port> -o coreaudio://<device_id>`
- **Nota**: O receiver deve ouvir em `0.0.0.0` (todas as interfaces) para aceitar conexões vindas do Sender remoto.

## 3. Implementação no Linux (`core/engine_linux.go`)
Atualmente apenas o `roc-sink` (envio) está implementado via PipeWire.

- **Novo Módulo**: `libpipewire-module-roc-source`.
- **Sintaxe**:
  ```bash
  pw-cli load-module libpipewire-module-roc-source \
    local.ip=0.0.0.0 \
    local.source.port=<port> \
    local.repair.port=<port> \
    local.control.port=<port> \
    source.name=Siren-Source \
    node.description="Siren Incoming Audio"
  ```

## 4. Orquestração e CLI
- **Manager**: Atualizar a lógica de criação de túnel padrão para permitir a escolha do modo.
- **CLI**: Adicionar uma flag ou argumento no comando de túnel:
  - `go run main.go tunnel start <id> --mode receiver`
